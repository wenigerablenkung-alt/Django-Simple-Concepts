⚙️ Middleware Pipeline

Every request passes through the middleware stack. Check your terminal to see the logging middleware in action!

🔄 The Middleware Pipeline (Visual)
  Browser Request
       │
       ▼
  ┌─────────────────────────────────┐
  │  SecurityMiddleware             │  ← Adds HTTPS headers
  ├─────────────────────────────────┤
  │  SessionMiddleware              │  ← Loads session from DB

  ├─────────────────────────────────┤

  └─────────────────────────────────┘
       │
       ▼
    Your VIEW function
       │
       ▼
  (Response travels back UP through middleware in reverse)
📋 Our Custom Middleware Code
class RequestLoggingMiddleware:
    def __init__(self, get_response):
        # Called ONCE at startup
        self.get_response = get_response

    def __call__(self, request):
        # ── BEFORE the view ──
        start = time.time()
        user = request.user.username

        # Call next middleware / view
        response = self.get_response(request)

        # ── AFTER the view ──
        ms = (time.time() - start) * 1000
        print(f"[{response.status_code}]"
              f" {request.path} {ms:.1f}ms")

        # Add custom header to response
        response['X-Processing-Time'] = f"{ms:.1f}ms"
        return response
🛡️ Security Headers Added By Middleware

These headers are on every response:

HeaderPurpose
X-Content-Type-OptionsPrevents MIME sniffing
X-Frame-OptionsBlocks clickjacking
Referrer-PolicyControls referrer info
